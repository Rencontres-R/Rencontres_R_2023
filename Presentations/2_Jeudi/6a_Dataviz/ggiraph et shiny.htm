ggiraph et shiny

ggiraph

Le package ‘ggiraph’ permet de créer des graphiques dynamiques et interactifs :

  • dynamiques avec des animations et des infobulles dans des documents HTML produits par Quarto ou ‘R Markdown’,
  • interactifs dans les applications “shiny” où les interactions avec le graphique ggiraph sont disponibles sous forme de valeurs réactives.

Le package ‘ggiraph’ est une extension de ‘ggplot2’, vous pouvez utiliser vos connaissances du package ‘ggplot’, pour rendre une forme interactive, utilisez les fonctions ‘ggplot’ mais suffixez leurs noms par _interactive.

Le package est aussi un vrai graphique R (au format SVG), vous devriez donc (si les dimensions sont les mêmes) obtenir la même sortie qu’avec votre IDE R favori.

Motivations

Vous souhaitez fournir à vos lecteurs plus d’informations que les informations visuelles usuelles d’un graphique. Vous souhaitez lorsque la souris de l’utilisateur est sûr ou à côté d’un élément graphique :

  • afficher une infobulle
  • animer le ou les éléments

OU Vous souhaitez permettre aux utilisateurs d’une application ‘shiny’ de sélectionner des éléments graphiques ; par exemple, vous pouvez rendre les points d’un diagramme de dispersion sélectionnables et disponibles en tant qu’input réactive.

Utilisé dans une application ‘shiny’, ‘ggiraph’ permet de travailler avec les interactions de l’utilisateur sur les éléments du graphique, les légendes, les titres et les éléments du thème ggplot ; la sélection et les hover sont disponibles en tant que valeur réactive.

Couverture de ggplot2

Tout ou presque :

  • 50 fonctions de type geom
  • 98 fonctions de type scale!
  • quelques fonctions de type guide
  • element_line_interactive, element_rect_interactive, element_text_interactive pour rendre les thèmes interactifs

Exemple

library(ggiraph)
p <- ggplot(dsmall, aes(x = price, y = depth, color = clarity)) 

p <- p + geom_point_interactive(
    aes(tooltip = color, data_id = color), 
    size = 3,
    hover_nearest = TRUE)

p <- p + scale_color_viridis_d_interactive(
  data_id = function(breaks) {
    format(breaks)
  },
  tooltip = function(breaks) {
    paste0("ceci est la légende pour ", breaks)
  },
  guide = "legend")

Exemple

```{r fig.width=8, fig.height=4, echo = FALSE}
girafe(ggobj = p, options = list(
  opts_hover(css = "stroke:black;"),
  opts_hover_inv(css = "opacity:.15;"),
  opts_toolbar(saveaspng = TRUE, pngname = "ggiraph", position = "topright")
))
```

Les trois aesthetics supportés par ggiraph ne sont pas des aesthetics supportés par ggplot2. C’est une fonctionnalité implémentée dans le package et elle n’est supportée que si la fonction girafe() est appelée.

  • tooltip : tooltip HTML à afficher
  • data_id : identifiant pour l’animation et pour shiny
  • onclick : JavaScript à exécuter lors du click (rarement utilisé)

Quelques remarques

Caution

Par défaut, la taille du graphique est automatiquement ajustée à 100% de la largeur de son conteneur web. Le graphique sera redimensionné si son conteneur est redimensionné. Ce comportement peut être contrôlé et annulé.

Note

Utilisez hover_nearest = TRUE pour appliquer l’effet de survol sur l’élément le plus proche lors du déplacement de la souris et non sur exactement l’élément. Dans ce cas, vous devez aussi fournir un paramètre data_id.

Fonctionnement shiny

Le graphique est transformé en object réactif et trois input réactifs sont mis à disposition, la sélection des éléments du panneau, la sélection des éléments de la légende et la sélection des éléments du thème.

Les éléments associés au data_id peuvent être sélectionnés et la sélection (la valeur du data_id) est disponible (dans le client et) le serveur de l’application.

Comment accèder aux sélections

Si le girafeOutput() a comme valeur d’identifiant girafeid, les valeurs réactives seront alors :

  • pour la sélection des panneaux, les éléments sélectionnés seront dans la valeur réactive input$girafeid_selected ou input$girafeid_hovered
  • pour la sélection des éléments de la légende dans la valeur réactive input$girafeid_key_selected ou input$girafeid_key_hovered
  • pour la sélection des éléments du thème dans la valeur réactive input$girafeid_theme_selected ou input$girafeid_theme_hovered.

Caution

Toutes ces sélections ne seront disponibles que si un aesthetic data_id.
